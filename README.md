# Bitrix Docker Environment

Docker-окружение для разработки и запуска проектов на **1С-Битрикс**, предназначенное в качестве альтернативы классической **BitrixVM**. В состав входят PHP-FPM, Nginx, MariaDB, Redis и отдельный контейнер для выполнения cron-задач.

## Особенности

- окружение полностью построено на Docker Compose;
- отсутствует зависимость от BitrixVM;
- все сервисы изолированы в отдельных контейнерах;
- конфигурации максимально приближены к рекомендациям Bitrix;
- проект ориентирован на локальную разработку и может использоваться как основа для production-инфраструктуры.

## Возможности

- PHP-FPM
- Nginx
- MariaDB 12
- Redis 8
- отдельный контейнер для cron-задач
- поддержка нескольких окружений (`dev`, `prod`)
- отдельные конфигурации PHP и Nginx для каждого окружения
- поддержка SSL-сертификатов для локальной разработки
- bind mount директории проекта
- постоянное хранение данных MariaDB и Redis

## Требования

- Docker Engine 24+
- Docker Compose v2

## Описание

Проект создавался как альтернатива **BitrixVM**, позволяющий запускать веб-сайты на **1С-Битрикс** полностью в Docker.

Конфигурации серверного ПО максимально приближены к рекомендуемым настройкам Bitrix и основаны на [официальной документации](https://dev.1c-bitrix.ru/learning/course/?COURSE_ID=32&LESSON_ID=32922&LESSON_PATH=3903.4862.20866.32912.32922) с небольшими доработками и исправлениями.

Проект не является универсальным решением. Конфигурации намеренно оставлены минимальными и рассчитаны на дальнейшую адаптацию под требования конкретного проекта.

Использование Docker значительно упрощает развёртывание приложений, перенос окружений между серверами, масштабирование сервисов и организацию процессов CI/CD. Кроме того, среда разработки может быть максимально приближена к продакшену без необходимости использования виртуальных машин.

По умолчанию в `.gitignore` исключены только ядро Битрикса и директория `upload`. При необходимости список следует дополнить файлами и каталогами, специфичными для проекта (например, `vendor`, пользовательскими модулями, локальными конфигурациями и т.п.).

## Настройка и запуск

### Настройка окружения

На основе файла `.env.example` необходимо создать файл `.env` и указать параметры окружения, включая домен локальной разработки.

### Генерация SSL-сертификатов для локальной разработки

> Для продакшен-окружения данный раздел не требуется.

В файле `certs/openssl.cnf` необходимо указать локальный домен в поле `DNS.2`.

```bash
openssl genrsa -out ./docker/certs/server.key 2048

openssl req \
    -x509 \
    -nodes \
    -days 3650 \
    -newkey rsa:2048 \
    -keyout ./docker/certs/server.key \
    -out ./docker/certs/server.crt \
    -config ./docker/certs/openssl.cnf
```

### Настройка отправки почты

Для настройки отправки почты используется `msmtp`.

Необходимо создать файл `docker/php/msmtprc`. В качестве основы можно использовать пример конфигурации:
https://marlam.de/msmtp/msmtprc.txt

## Управление

```bash
docker compose up -d        # сборка и запуск контейнеров

docker compose stop         # остановка контейнеров

docker compose down         # остановка и удаление контейнеров

docker compose logs -f      # просмотр логов

docker compose exec php sh  # вход в PHP-контейнер
```

## Установка 1С-Битрикс

Способ установки зависит от того, создаётся новый проект или переносится существующий сайт.

### Новый проект

В каталог `www` необходимо поместить файл [bitrixsetup.php](https://www.1c-bitrix.ru/download/scripts/bitrixsetup.php), после чего открыть страницу:

https://bitrix.localhost/bitrixsetup.php

### Перенос существующего проекта

1. Подготовить резервную копию сайта и базы данных. При создании резервной копии необходимо **отключить** параметр «Архивировать публичную часть».
2. В каталог `www` необходимо поместить файл [restore.php](http://www.1c-bitrix.ru/download/scripts/restore.php)
3. Открыть https://bitrix.localhost/restore.php
